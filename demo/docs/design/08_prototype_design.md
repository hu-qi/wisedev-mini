# 08 原型实现设计（Prototype Design）

适用对象：国央企 / 政企用户（强调可演示、可交接、可继续开发；遵循分层、契约驱动、切片开发）

## 1. 原型范围与页面清单
### 1.1 页面清单（占位）
- 登录/统一身份对接占位页（如适用）
- 首页/工作台
- 业务办理：列表/详情/表单
- 审批中心：待办/已办/详情
- 管理后台：组织/权限/字典占位
- 审计查询：检索/详情/导出占位

## 2. 目录结构与分层约定
建议结构（可按实际工程微调）：

- `src/views/`：页面与路由入口（按模块/切片划分）
- `src/services/`：接口调用封装（按 OpenAPI 契约拆分）
- `src/types/`：TypeScript 类型（与契约/数据模型一致）
- `src/mocks/`：mock handlers 与 mock 数据（契约驱动）
- `src/router/modules/`：路由模块（按业务模块拆分）

## 3. 路由规划（占位）
| 路由 | 页面 | 访问控制 | 备注 |
| --- | --- | --- | --- |
| /workbench | 工作台 | 登录后 | - |
| /biz/list | 业务列表 | 角色/数据范围 | - |
| /audit/query | 审计查询 | 审计角色 | 导出占位 |

## 4. 状态与交互基线
- 每个页面必须具备：loading / empty / error / success
- 表单必须具备：必填/格式/范围/枚举校验与明确提示语
- 列表必须具备：分页（前端分页或 mock 分页占位）

## 5. 权限与审计（原型约束）
### 5.1 权限
- 菜单/按钮权限：基于角色进行渲染控制（占位实现）
- 数据范围：按机构/部门/本人控制（占位实现）

### 5.2 审计留痕
- 点位：登录/查询/导出/新增/修改/提交/审批
- 字段：操作人、时间、对象、动作、结果、错误码（如有）

## 6. mock 切真实接口约定
- `services` 层保留真实接口替换点位
- `mocks` 仅在 mock 模式启用（由 `.env.mock` 或环境变量控制）
