# 验收标准（Acceptance）

适用对象：国央企 / 政企用户（强调合规、安全、可追溯、可交接）

## 1. 总体说明
本项目以“可运行、可预览、可演示、可交接”为最低验收基线；原型阶段可使用 mock，但接口契约、字段口径、权限与审计结构必须预留并可验证。

## 2. 功能验收
### 2.1 关键流程可演示
- [ ] 经办人核心流程可完整闭环（发起/办理/提交/查询）
- [ ] 负责人审批流程可完整闭环（同意/驳回/转办/加签占位）
- [ ] 管理员配置流程可完整演示（组织/权限/字典/参数占位）
- [ ] 审计查询可完整演示（按条件检索、查看详情、导出占位）

### 2.2 权限与数据范围
- [ ] 未登录访问被拦截（如适用）
- [ ] 角色权限矩阵生效（菜单、按钮、接口占位）
- [ ] 数据范围控制生效（部门/机构/本人数据占位）

### 2.3 交互与状态
- [ ] 列表/详情/表单具备 loading/empty/error/success 状态
- [ ] 表单校验清晰（必填、格式、范围、枚举）
- [ ] 异常提示可读、可定位（建议包含错误码/提示语规范）

## 3. 数据与契约一致性
- [ ] 页面字段与数据模型一致（字段名、类型、枚举、单位）
- [ ] mock 数据与 OpenAPI 契约一致（请求/响应结构一致）
- [ ] 核心接口具备示例（示例值符合政企业务口径与敏感信息要求）

## 4. 安全与合规（原型基线）
- [ ] 关键操作具备审计留痕点位说明（原型可为占位实现）
- [ ] 敏感字段展示策略明确（脱敏/隐藏/权限控制说明）
- [ ] 文档中明确内网/专网环境运行假设与依赖获取方式

## 5. 工程交付验收
- [ ] `npm install` 可完成依赖安装（如需离线镜像/私服，提供说明）
- [ ] `npm run dev` 可启动并可访问核心页面
- [ ] `npm run build` 与 `npm run preview` 可通过
- [ ] 提供运行与演示路径说明（README 或部署指南）
