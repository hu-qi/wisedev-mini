# 05 数据模型与口径（Data Model）

适用对象：国央企 / 政企用户（强调字段口径一致、可追溯、可审计、敏感数据保护）

## 1. 数据口径总览
### 1.1 主数据与字典
- 组织机构（Org）
- 人员/账号（User）
- 角色与权限（Role/Permission）
- 字典与枚举（Dictionary/Enum）

### 1.2 口径原则
- 字段命名、单位、枚举值必须唯一且可追溯
- 与页面展示、导出报表、接口契约保持一致
- 敏感字段需标注脱敏策略与访问权限

## 2. 实体定义（建议按实体逐段编写）
### 2.1 实体：组织机构（Org）（占位）
| 字段 | 类型 | 是否必填 | 说明 | 示例 | 敏感级别 |
| --- | --- | --- | --- | --- | --- |
| orgId | string | 是 | 机构 ID | \"1001\" | 一般 |
| orgName | string | 是 | 机构名称 | \"XX 分公司\" | 一般 |

### 2.2 实体：用户账号（User）（占位）
| 字段 | 类型 | 是否必填 | 说明 | 示例 | 敏感级别 |
| --- | --- | --- | --- | --- | --- |
| userId | string | 是 | 用户 ID | \"u001\" | 一般 |
| mobile | string | 否 | 手机号 | \"138****0000\" | 敏感（脱敏展示） |

### 2.3 实体：业务对象（XXX）（占位）
（按实际业务补充字段、状态、枚举）

## 3. 枚举与字典
### 3.1 枚举定义（占位）
- 状态枚举：DRAFT / SUBMITTED / APPROVED / REJECTED
- 权限范围：SELF / DEPT / ORG / ALL

### 3.2 字典来源与管理方式（占位）
- 来源：配置文件 / 后台字典服务（原型可 mock）
- 版本：字典变更记录与影响说明（占位）

## 4. 与契约/页面的映射关系
- OpenAPI：字段/枚举需同步到 `openapi.yaml`
- 页面：表单字段/列表字段需引用本文件口径
- mock：mock 数据必须符合字段类型与枚举值

## 5. 审计与合规字段（占位）
- 操作人、操作时间、操作类型、对象标识、结果
- 数据变更前后摘要（占位）
