# 06 概要设计（Solution Outline）

适用对象：国央企 / 政企用户（内网 / 专网 / 私有云优先，强调合规、安全、可运维、可追溯）

## 1. 总体架构概述
- 系统形态：前端原型（Vue） + mock（契约驱动）
- 运行环境：内网/专网开发与演示优先（离线依赖与镜像策略需说明）
- 分层原则：视图层（views）/ 服务层（services）/ 类型层（types）/ mock（mocks）/ 路由（router）

## 2. 组件与模块划分
### 2.1 页面模块（占位）
- 模块 A：（例如台账管理）
- 模块 B：（例如流程审批）
- 模块 C：（例如组织与权限 / 审计留痕）

### 2.2 公共能力（占位）
- 权限控制：菜单/按钮/数据范围（原型可做占位实现，但需预留结构）
- 审计留痕：关键操作留痕点位说明（原型可做占位实现）
- 字典/枚举：统一口径与可配置来源（占位）
- 错误处理：错误码、提示语规范、异常兜底页（占位）

## 3. 技术选型与约束
- 前端框架：Vue 3 + TypeScript + Vite
- 接口契约：OpenAPI 3.0（`openapi.yaml` 为唯一事实来源）
- mock 策略：基于契约的 mock（请求/响应结构对齐）
- 兼容性约束：国产浏览器 / 兼容模式要求（占位）
- 国产化约束：国产 OS / 软硬件环境适配要求（占位）

## 4. 部署与交付策略（原型阶段）
### 4.1 本地运行与预览
- `npm run dev`：开发与演示
- `npm run build`：构建验证
- `npm run preview`：预览验证

### 4.2 环境边界与依赖获取
- 内网/专网依赖：私服/镜像源配置说明（占位）
- 配置文件：`.env.mock`（说明 mock 开关、后续切真实接口约定）

## 5. 风险与控制措施（政企重点）
- 合规风险：权限/审计未覆盖 → 通过结构预留与文档约束降低偏差
- 口径漂移：字段/枚举不一致 → 以数据模型与 OpenAPI 统一约束
- 交付不可复现：环境依赖缺失 → 强制输出运行与演示说明
